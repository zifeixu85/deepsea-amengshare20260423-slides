## Users

### 产品叙事层（虚构产品 DeploySea 的用户）
独立开发者 / AI-native builder：本地 Demo 已经能跑，卡在"从 Demo 到产品"的上线、调试、维护阶段。他们**见过所有模板**，能一眼闻出 AI 味页面，没耐心读废话。

### 真实观众（直播观众）
DeepSea 课程一期及后续学员。他们已经会用 AI 写页面，但输出总有"demo 味 / AI 味"。他们会**把 v1 和 v3 并排看**，学习"做对了"到底长什么样。

**关键含义**：v3 必须与 v1 产生**一眼可见的视觉差距**，不是微调，是换一种思路。

## Brand Personality

三个词：**有主张的、已上线的、工业质感的**（opinionated / shipped / industrial）

**不是**：友好、活泼、欢快、技术炫技

**感觉**：像读一篇印在好纸上的技术随笔——自信、安静、精确。

**情绪目标**：观众看到 v3 要觉得"这是一个有人在认真维护的真产品"，不是"这又是一个模板"。

## Aesthetic Direction

### 主方向
**Bento 网格**：Apple 发布会那种多区块拼接构图。每个 cell 尺寸、权重、内容类型都不一样——**没有重复的 3 卡行**，**没有图标-标题-正文的模板网格**。

### Theme
**Light**（暖奶白 / 纸感底色）。

原因：产品是桌面工具，在亮环境使用；深色 Bento 容易滑向 AI 霓虹套路，亮色 Bento 更像编辑排版。

### Anti-References（明确要避开）
- v1 自身——烂大街蓝色 SaaS 模板（这是反面教材）
- Vercel / Linear 克隆版的深色 + 渐变光球
- AI 套路配色：深背景 + 青 + 紫蓝渐变
- 玻璃拟态装饰性用法
- 图标-圆角卡片-阴影三件套

### Color
- **禁止蓝色**（包括淡蓝、靛蓝、青蓝任何变体）
- 中性色**略带品牌色调**（chroma 0.005–0.01 的 warm tint）
- **单一强调色**，按 60-30-10 稀用。候选方向（实现时从中选一个并坚持到底）：
  - Ember / 赤陶（偏暖的强红橙）
  - Deep olive / 苔绿（冷静、工业）
  - Bold red-orange / 朱砂（有主张、切入感强）
- 纯黑、纯白**禁用**，使用带色相倾向的近黑近白。
- 用 `oklch()` 定义，不是 hex / hsl。

### Typography
**封禁列表**（来自 impeccable skill，坚决不用）：
Inter, Roboto, Arial, Open Sans, DM Sans, DM Serif, Plus Jakarta Sans, Outfit, IBM Plex 全家族, Space Mono, Space Grotesk, Instrument Sans, Instrument Serif, Fraunces, Newsreader, Lora, Crimson 全家族, Playfair Display, Cormorant 全家族, Syne。

**方向**：一套 distinctive display + 一套 humanist body sans + 可选一套 mono（用于 CLI / 版本号小字）。

**候选**（实现时再敲定，优先 Google Fonts 可直链）：
- Display：Funnel Display / Bricolage Grotesque / Boldonse / Redaction
- Body：Host Grotesk / Geist
- Mono：JetBrains Mono / Geist Mono

字号规则：heading 用 `clamp()` 流体，5 档对比（比例 ≥ 1.25）。body 不超过 75ch。

### Composition（Bento 具体打法）
- 首屏不是居中标题 + CTA，而是**偏置的主 cell + 旁边对话性副 cell**
- 至少 **3 种 cell 宽度**（2/3 跨度、半宽、1/3 跨度），至少 **2 种 cell 高度**
- 允许一个 cell 做**视觉主角**（大字 / 特殊底色 / 工业感插画），其余 cell 克制
- 不要全圆角，Bento 的辨识度来自**统一圆角 + 统一描边 + 差异化填充**
- 允许"破格"：跨列的 quote cell、半透底的 code cell

### Motion
- 首屏一次**编排过的进场**（stagger reveal），之后克制
- 只动 `transform` 和 `opacity`
- 缓动用 ease-out-expo，不用 bounce / elastic
- 尊重 `prefers-reduced-motion`

## Design Principles（来自 A梦 大纲第六部分，所有决策必须过）

1. **说清楚 > 好看**：每一段 copy 先回答用户疑问，不做装饰性标题。
2. **首屏只说一件事**：主标题一句话讲清价值，CTA 具体不说"Get Started"。
3. **视觉分层，不是视觉热闹**：一个区域只有一个视觉主角。
4. **结构即说服顺序**：Bento cell 的顺序 = 论证顺序（看懂 → 相关 → 信任 → 行动）。
5. **不默认模板**：每个 cell 是刻意构图，不是掉进去的卡片。
6. **桌面优先 + 手机必须能用**：桌面是演示主屏；手机端 Bento **塌成单列的垂直叙事**，而不是缩小版桌面。

---

## Responsive

- **Primary**：桌面大屏（直播投屏），1440px+ 测试基准
- **Secondary**：1024px / 768px / 375px 都要无溢出、无断版
- 断点不是通用 px，是**内容驱动**：哪里视觉坍塌了就在那里断
- 手机端 Bento：单列，cell 按论证顺序垂直排，视觉主角 cell 保持但高度压缩

---

## Accessibility

- WCAG AA 对比度底线
- 所有交互 focus-ring 明显
- 尊重 reduced-motion
- 语义标签优先（`<main>` / `<section>` / `<article>`），不用 `<div>` 堆
